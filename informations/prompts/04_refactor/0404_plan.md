## TASK
Write a complete refactor plan:

### 1. Structural Delta
State explicitly:
- Current structure (before)
- Target structure (after)
- What is identical in both

### 2. Migration Strategy
Only required if `ran_0403_scope.md` flags any MIGRATION REQUIRED items.
State which strategy and why:
- **In-place migration** — transform existing data during execution
- **Parallel support** — support both old and new formats temporarily
- **Clean break** — old data incompatible, requires version bump

If no migration flags exist: write NOT APPLICABLE

### 3. Phase Breakdown
Each phase must be independently shippable.
A broken phase must not break the previous phase.

For each phase:
- **Phase name**
- **What it restructures**
- **Migration step** — if this phase touches a persisted key, field name,
  or export format, state the migration action here explicitly.
  This step executes inside 0405 as part of this phase.
  If no migration: write NONE.
  Migration actions must follow the strategy declared in §2.
- **What it leaves temporarily inconsistent**
  (and which phase resolves it)
- **Rollback cost if this phase fails:** LOW / MEDIUM / HIGH
- **Hard stop triggers for this phase**
- **Verification** — what the user does to confirm this phase worked,
  in plain language. Must match what was actually built:
  - Structural phase → "open the app and confirm [specific thing] still works"
  - Migration phase → "export a record and confirm it opens correctly in [tool]"
  - Consolidation phase → "make a change in [one place] and confirm [effect] appears"
  - Never write a verification the user cannot perform themselves

## CONSTRAINT
- Every phase must have a rollback path
- Migration steps belong inside the phase that triggers them — not as a
  separate phase unless the migration is large enough to be standalone
- If any phase unavoidably breaks a behavioral invariant temporarily,
  it must be the shortest possible phase and the next phase must
  restore it immediately
- If the plan requires breaking a hard stop condition,
  flag it as BLOCKER — do not route around it
- Do not add features inside a refactor plan
- Do not write implementation code — describe what changes and why
