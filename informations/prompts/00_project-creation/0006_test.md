Write tests that cover:

1. **Happy path** — expected inputs produce expected outputs
2. **Edge cases** — empty arrays, null values, missing fields
3. **Failure cases** — what should break and how it should break
4. **Data integrity** — does the output match the data model from Plan §4?

Format:
- Each test must print PASS or FAIL with a description
- Group tests by function name
- End with a summary: X passed, Y failed
