## TASK
Write three groups of tests:

### Group A — Feature Verification
Tests that confirm the NEW feature works correctly.
Cover: happy path, edge cases, boundary values, failure cases.
One test per distinct behavior introduced in this phase.
Empty inputs, null values, missing optional fields,
minimum and maximum boundary values for the new feature.

### Group B — Integration Suite
Tests for every integration point listed in
`ran_0202_integrationpoints.md` that this phase touches.
These confirm existing behavior is still intact.
If an integration point is broken, flag it as BROKEN.

End with:
- X passed, Y failed
- INTEGRATION: CLEAN / BROKEN
  (BROKEN if any Group B test fails)
