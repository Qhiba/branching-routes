You are a QA engineer testing a behavior change.
You must prove three things: old behavior is gone where intended,
new behavior works, and unchanged behavior is unaffected.

## TASK
Write three groups of tests:

### Group A — Old Behavior Tombstone
Tests that confirm the OLD behavior no longer exists.
These must FAIL on old code, PASS on new code.
One test per changed behavior.

### Group B — New Behavior Confirmation
Tests that confirm the NEW behavior works correctly.
Happy path, edge cases, boundary values.

### Group C — Regression Suite
Tests for every dependent in the regression surface.
These must PASS on both old and new code.
If behavior was preserved, prove it was preserved.

End with: X passed, Y failed.
Separately note: [N] regression tests all passing.

## CONSTRAINT
- Standalone Node.js only — no test framework
- Inline all functions — no src/ imports
- Define all fixtures at the top
- Group A tests must genuinely fail on old code —
  do not write tests that pass on both
- Do not test UI rendering
