## Issue 9: Node drag teleports to cursor on release instead of dragging smoothly
- **File:** `src/components/GraphCanvas.jsx`
- **Root Cause:** `reactFlowNodes` was fully derived via `useMemo` from the Zustand store. React Flow updates node positions internally on every drag frame, but those updates were discarded because each render re-derived positions from the unchanged store. The store only updates on `dragStop`, so the node appeared frozen mid-drag and snapped on release.
- **Changed Section:**
```javascript
// Local state that React Flow owns during drag
const [rfNodes, setRfNodes] = useState(derivedNodes);
const isDragging = useRef(false);

// Sync from store only when NOT dragging
useEffect(() => {
  if (!isDragging.current) {
    setRfNodes(derivedNodes);
  }
}, [derivedNodes]);

// Let React Flow apply all changes (position moves, selection, etc.) locally
const onNodesChange = useCallback((changes) => {
  setRfNodes(nds => applyNodeChanges(changes, nds));
}, []);

const onNodeDragStart = useCallback(() => {
  isDragging.current = true;
}, []);

const onNodeDragStop = useCallback((event, node) => {
  isDragging.current = false;
  updateNode(node.id, { position: node.position }); // persist final position to store
}, [updateNode]);
```
- **Explanation:** Introduced a local `rfNodes` state owned by React Flow for mid-drag position updates. The store is only written to on `dragStop` (AR-03/AR-05 compliant). An `isDragging` guard prevents the store-sync `useEffect` from overriding React Flow's in-progress drag positions.
