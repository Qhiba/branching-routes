# ran_0202_datamodelimpact.md — Data Model Impact

**Feature:** Variants_on_nodes_and_Options_on_choices
**Generated:** 2026-04-18

---

## New fields added to existing entities

### `node.data` on common nodes — new field: `variants[]`

```json
"variants": [
  {
    "id": "v-{uuid}",
    "label": "Variant label shown to designer",
    "text": "Alternate narrative text shown to reader",
    "requires": {
      "operator": "and",
      "conditions": [
        { "flag": "f-{uuid}", "state": true }
      ]
    }
  }
]
```

- **Absent = `[]`** — all consumers must default via `?? []` or `|| []`. No crash on legacy nodes.
- `requires` is the identical `{ operator, conditions[] }` object already used by edges. No new schema needed.

### `node.data` on choice nodes — new field: `options[]`

```json
"options": [
  {
    "id": "opt-{uuid}",
    "label": "What the player sees as a selectable action",
    "requires": {
      "operator": "and",
      "conditions": []
    },
    "flags_set": ["f-{uuid}"],
    "status_set": [
      { "statusId": "sp-{uuid}", "amount": 1 }
    ]
  }
]
```

- **Absent = `[]`** — same defaulting rule.
- `flags_set[]` and `status_set[]` match the existing node-level schema already rendered by `NodeInspector`.

### `edge` — new optional field: `optionId`

```json
{
  "id": "e-{uuid}",
  "sourceId": "n-{uuid}",
  "targetId": "n-{uuid}",
  "label": "...",
  "condition": null,
  "optionId": "opt-{uuid}"
}
```

- **Absent or `null`** — edge behaves exactly as before. `EdgeInspector` shows "No option" when `optionId` is absent.
- Set by `GraphCanvas` at connect time if the source handle ID matches a known option ID.

---

## Is every addition strictly additive?

**Yes.** No existing field is renamed, removed, or given a new type. All three new fields (`variants`, `options`, `optionId`) are optional and absent-tolerant.

---

## Export/import round-trip survival

- `exportGraph()` in `narrativeStore` serialises `common`, `choice`, `ending`, and `edges` verbatim. Because `variants` and `options` live inside `node.data`, and `optionId` lives on edges, they are serialised automatically with no store changes required.
- **Import:** `fileSystem.js` passes node data through unchanged. Consumers default absent arrays to `[]`. Existing v1–v4 files import without modification.
- **Schema version decision:** A bump to `schemaVersion: 5` is **recommended but not required** for correctness. It is required only if the team wants to make "no options on a choice node" detectable as a pre-feature file vs. an authored empty-option node. **This decision is made in Phase 1; it does not block execution.**

---

## ID prefixes touched

| Prefix | Entity | Status |
|---|---|---|
| `v-` | Variant ID (new) | New prefix, generated by `generateId('v')` |
| `opt-` | Option ID (new) | New prefix, generated by `generateId('opt')` |
| `n-`, `e-`, `f-`, `sp-`, `p-`, `c-` | All existing | Unchanged |

The `opt-` prefix for option IDs also serves as the React Flow `handleId` for per-option source handles on `ChoiceNode`. The prefix ensures no collision with any node, edge, or flag ID.
