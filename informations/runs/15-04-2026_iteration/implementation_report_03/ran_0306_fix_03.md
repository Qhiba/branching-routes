# Fix Report: Phase 3

## Addressed Items
- **Human Note 1** ("The `set start node` button are not giving a node start node value"): Fixed the simulation crash caused by `simulationStore.js` attempting to map the deprecated `flags` array when extracting variables for `start()`. The uncaught `TypeError` was captured by `TopBar`'s global start-simulation `catch` block, which unconditionally surfaced the error message `"Set a Start Node first"`. This led to the symptom of the "set start node" button appearing unresponsive when attempting to start the simulation.

## Corrected Code Sections

**`src/store/simulationStore.js`**
```javascript
    const initialFlags = {};
    if (graphState.flag) {
      Object.values(graphState.flag).forEach(f => {
        initialFlags[f.id] = f.state;
      });
    }
    if (graphState.status) {
      Object.values(graphState.status).forEach(s => {
        initialFlags[s.id] = s.value;
      });
    }
```
- **What was fixed:** Replaced the legacy `graphState.flags.forEach()` iteration with separate loops to extract values from `graphState.flag` and `graphState.status` dictionaries.
- **Impact check:** This pulls forward a minor piece of the Phase 4 behavior delta (simulation data extraction) to enable `start()` without crashing, but it does not violate the architecture invariants or preservation lists.
