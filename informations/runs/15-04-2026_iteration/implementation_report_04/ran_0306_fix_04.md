### Issue 1: Error occurred when clicking the node (React maximum update depth exceeded)
```javascript
  const nodeType = useNarrativeStore(state => {
    if (!selectedNodeId) return undefined;
    if (state.common[selectedNodeId]) return 'common';
    if (state.choice[selectedNodeId]) return 'choice';
    if (state.ending[selectedNodeId]) return 'ending';
    return undefined;
  });

  const node = useNarrativeStore(state => {
    if (!selectedNodeId) return undefined;
    if (state.common[selectedNodeId]) return state.common[selectedNodeId];
    if (state.choice[selectedNodeId]) return state.choice[selectedNodeId];
    if (state.ending[selectedNodeId]) return state.ending[selectedNodeId];
    return undefined;
  });
```

**Fix Description:** Split the unified Zustand state selector (which was returning a fresh inline object literal `{node, nodeType}` on every evaluation) into two separate flat selectors to prevent unstable selector references from failing React's `getSnapshot` consistency check and triggering infinite render loops.
**Impact:** This affects **neither** the behavior delta nor the preservation list, as it strictly fixes the technical execution pattern of the previously authorized lookup while maintaining identical state retrieval behavior.
