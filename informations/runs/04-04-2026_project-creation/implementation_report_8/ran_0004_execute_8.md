## Summary

Phase 8 implements the primary interaction model for Branching Routes V2: context-sensitive right-click menus and global keyboard shortcuts. Users can now create, delete, duplicate, and manipulate entities without any sidebar or toolbar — all interaction flows through the context menu and keyboard.

## Design Decisions

1. **`deleteKeyCode` set to `null`**: React Flow's built-in delete key handling would conflict with our custom keyboard shortcut handler. We disable it and handle Delete entirely through `useKeyboardShortcuts.js` for consistent behavior.

2. **Menu items as static arrays**: Canvas, node, and edge menu items are defined as static constants in `useContextMenu.js` rather than being dynamically generated. This keeps the logic simple and predictable for the current phase. Future phases can make them dynamic if needed.

3. **Viewport boundary clamping**: The context menu uses a `useEffect` to reposition itself if it would overflow the viewport edges. This ensures the menu is always fully visible.

4. **Duplicate creates shallow copy**: Node duplication copies basic fields (name, type, chapter, path, description) with an offset position, but does NOT deep-clone `next[]`, `requires`, `variants`, or `options`. This prevents creating broken references to other entities and follows a "duplicate the structure, not the connections" pattern. This matches typical graph editor behavior (e.g., Figma duplicates the visual object, not the connections).

5. **Escape priority chain**: Escape key follows a priority chain: context menu → command palette → inspector → deselect. Only the highest-priority open element is dismissed per keypress.
